# 🎬 Movie Archive Requirements

## 📌 문서 소개

Movie Archive는 사용자가 감상한 영화를 체계적으로 기록하고, 단순한 기록을 넘어 타인과 취향을 공유할 수 있는 커뮤니티형 영화 아카이브 서비스입니다. 본 문서는 시스템이 제공해야 할 기능적/비기능적 요구사항을 정의하여 개발 방향성을 명확히 하고, 팀원 간의 효율적인 협업을 위한 토대를 마련하는 데 목적이 있습니다.

## 2. 시스템 설계 (movie-archive)

### 2.1 아키텍처 다이어그램

```
[사용자] -> [View / UI] -> [Controller] -> [Service] -> [DB]
```

- **사용자(User)**: 영화 검색/저장/리뷰 작성/내 아카이브 관리
- **View/UI**: 웹(또는 앱) 화면에서 목록/상세/작성/마이페이지 제공
- **Controller**: 요청 라우팅, 입력 검증, 인증 상태 확인, 응답 포맷(JSON/HTML) 구성
- **Service**: 핵심 비즈니스 로직(아카이브 저장, 리뷰/평점, 태그, 즐겨찾기 등)
- **DB**: 사용자/영화/아카이브/리뷰/태그 관계 데이터 영속화

---

### 2.2 데이터베이스 설계 (ERD 요약)

> movie-archive에서 “영화 자체 정보(movies)”와 “사용자별 저장(archives)”을 분리하면 확장/중복 방지에 유리합니다.

| 테이블명 | 주요 컬럼 |
|---|---|
| users | id, name, email, password, created_at |
| movies | id, title, original_title, release_date, runtime, overview, poster_url, created_at |
| archives | id, user_id, movie_id, status(watched/to_watch), watched_at, created_at |
| reviews | id, user_id, movie_id, rating, content, created_at |
| tags | id, name, created_at |
| archive_tags | id, archive_id, tag_id |

**관계(요약)**
- users(1) : archives(N)
- movies(1) : archives(N)
- users(1) : reviews(N)
- movies(1) : reviews(N)
- archives(N) : tags(N) 는 `archive_tags`로 다대다 매핑

### 2.3 화면 설계 (Wireframe)

필수 화면(예시)
- **메인 화면**
    - 최신/인기 영화(또는 내 최근 활동) 노출
    - 검색창 바로 제공
- **로그인/회원가입 화면**
    - 이메일/비밀번호 로그인
    - 회원가입(이름, 이메일, 비밀번호)
- **영화 목록 화면**
    - 검색 결과 목록
    - 필터(개봉년도, 장르 등) 또는 정렬(인기순/최신순)
- **영화 상세 화면**
    - 포스터/개요/개봉일/러닝타임
    - “내 아카이브에 추가(보고싶어요/봤어요)”
    - 리뷰/평점 작성 및 목록
- **내 아카이브(마이페이지)**
    - 보고싶어요/봤어요 탭
    - 내가 남긴 리뷰 모아보기
    - 태그로 분류/검색

- Figma, Kakao Oven, 직접 스케치 후 사진 첨부
